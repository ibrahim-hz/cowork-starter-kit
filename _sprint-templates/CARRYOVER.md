# Sprint <N> — Carryover to Sprint <N+1>

Only create this file if work was deferred. If nothing carries over, say so in `RETRO.md` and don't create this file.

## Why carryover exists

Avoiding a `.5` repair sprint. Small, well-understood leftovers get rolled into the next sprint's build plan; larger repair work still warrants a `.5` sprint of its own.

## What's carrying over

For each item:

### <short title>

- **What:** one-sentence description
- **Why deferred:** ran out of time / blocked on external dependency / scope cut by user / discovered mid-sprint
- **Where it left off:** last commit hash or branch, plus a one-line "state of the world"
- **What's needed to finish:** concrete remaining steps
- **Suggested placement in next sprint:** "first prompt" / "any time" / "depends on <other item>"
